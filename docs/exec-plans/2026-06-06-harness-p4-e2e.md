## 전략

운영 백엔드 없이 실제 UI 를 검증하기 위해, 프런트의 기존 폴백 설계를 그대로 활용:

- `scripts/serve-static.mjs` (제로 의존성 정적 서버) — `/api/*` 는 404 →
  프런트가 번들 데이터(`data/*.json`)로 폴백 (운영 코드의 실제 경로)
- 세션: `localStorage('itpe.session')` 시드 (클라이언트 게이트 통과)
- 스텁은 `/api/login` 단 1개 (로그인 흐름 테스트용) — 나머지는 실코드 그대로
- 서비스워커 차단(`serviceWorkers: 'block'`) — 테스트 간 캐시 간섭 제거
- 외부 폰트 CDN 차단 — 네트워크 비결정성 제거

## P5(flash.js 분할) 판단 — 보류

하네스 점검 결과 **지금은 분할하지 않는 것이 옳다**:

1. flash.js 는 단일 IIFE 클로저로 `state`·`els` 를 공유 — 클래식 스크립트 분할은
   전역 네임스페이스 재설계가 필요한 **대수술** (동작 변경 위험 높음)
2. 현재 E2E 커버리지는 3흐름뿐 — TTS·타이머·스와이프·관리자 도구·인쇄 등
   미커버 영역이 많아 분할의 안전망이 부족
3. "동작 임의 변경 금지" 제약상, 리스크 대비 즉시 효익이 없음

**재개 조건**: flash.js 의 특정 영역을 크게 수정하는 기능 요청이 생길 때,
그 영역에 E2E 를 먼저 추가 → 해당 영역만 점진 분리. 내부 지도는
[../../ARCHITECTURE.md](../../ARCHITECTURE.md) 'flash.js 내부 지도'가 제공.
